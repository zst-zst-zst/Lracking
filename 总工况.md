## 二、反制机制与任务目标

### 2.1 反制触发条件

**仅在对方空中机器人发起空中支援时**，己方激光发射装置及执行机构(云台)才上电。

### 2.2 被瞄准进度 P 的积累

- **未被照射**: P 以 0.5/s 匀速衰减, t和n立即归零
- **被照射**: P 停止衰减, 每 0.1s(Δt)触发一次进度增加:
  - 第1个0.1s: P += 1
  - 第2个0.1s: P += 2
  - 第n个0.1s: P += n
- 照射不足 0.1s 中断: t、n归零

**关键推论**: 连续照射效率远高于间断照射 (累计递增), 需要云台跟踪精度高、抖动小。

### 2.3 锁定效果与难度递增

| 锁定次数 | P0阈值 | 锁定效果 | 目标变化 |
|---------|--------|---------|---------|
| 第1次 | 50 | 发射机构锁定 45s | - |
| 第2次 | 100 | 发射机构锁定 45s | 检测区域缩减为 3/5 |
| 第3次 | 100 | 发射机构锁定 45s | 检测区域缩减为 1/5 且不再主动发光 |

每局最多反制 **3次**。

### 2.4 激光发射装置规格

| 参数 | 限制 |
|------|------|
| 功率 | < 1mW |
| 波段 | ≤ 730nm (建议 600-700nm) |
| 光斑直径 | 建议 ≥ 5mm |
| 安全等级 | Class I 或 Class II |
| 数量 | 仅 1 个 |

## 三、目标与精度预算

### 3.1 激光检测模块 (S190)

```
整体: 72mm(高) × 50mm(宽) × 50mm(深)
检测区域: φ42mm 圆环, 中空 φ20mm
必须打在正中心
```

### 3.2 各距离精度预算

| 距离 | 检测区φ42mm对应角度 | 1像素(fx=1000)对应 | 精度容差 |
|------|---------------------|-------------------|----------|
| 10m | 0.24° | 0.057° | ±0.12° |
| 20m | 0.12° | 0.057° | ±0.06° |
| 25m | 0.096° | 0.057° | ±0.048° |
| **30m** | **0.08°** | **0.057°** | **±0.04° ≈ ±0.7px** |

### 3.3 对跟踪系统的要求

| 指标 | 要求 | 来源 |
|------|------|------|
| 稳态角度误差 | < ±0.04° (30m) ~ < ±0.12° (10m) | 精度预算 |
| 连续照射时间 | 尽可能长 (递增P) | 反制机制 |
| 云台最大追踪速度 | > 200 deg/s | 目标快速运动时 |
| 系统延迟 | < 30ms (检测5ms + 通信3ms + 电控响应20ms) | 远距离跟踪 |

## 四、通信协议

### 4.1 NUC → STM32 (29字节, USART6)

```
[0x5A 0xA5] [mode 1B] [yaw 4B] [yaw_vel 4B] [yaw_acc 4B]
            [pitch 4B] [pitch_vel 4B] [pitch_acc 4B] [0x7F 0xFE]
```

| 字段 | 单位 | 坐标系 | 说明 |
|------|------|--------|------|
| mode | - | - | 0=不控制, 1=控制不开火, 2=控制且开火 |
| yaw | deg | IMU.YawTotalAngle | 绝对目标角度 |
| pitch | deg | -IMU.Pitch (取反) | 绝对目标角度 |
| yaw_vel | deg/s | 目标运动方向 | 速度前馈 (NUC可发0, STM32差分补算) |
| pitch_vel | deg/s | 目标运动方向 | 速度前馈 |
| yaw_acc | deg/s² | - | **当前不使用, 恒=0** |
| pitch_acc | deg/s² | - | **当前不使用, 恒=0** |

### 4.2 STM32 → NUC (43字节)

```
[0x5A 0xA5] [mode 1B] [q0~q3 16B] [yaw 4B] [yaw_vel 4B]
            [pitch 4B] [pitch_vel 4B] [bullet_speed 4B] [bullet_count 1B] [0x7F 0xFE]
```

| 字段 | 说明 |
|------|------|
| q[4] | 四元数 (q1,q2取反使NUC解算pitch一致) |
| yaw | YawTotalAngle (deg) |
| yaw_vel | Gyro Z (deg/s) |
| pitch | -IMU.Pitch (deg, 已取反) |
| pitch_vel | -Gyro Y (deg/s, 已取反) |

### 4.3 坐标系说明

```
YAW:   IMU 的 YawTotalAngle (度, 累积, 无限制)
PITCH: -IMU.Pitch (度, 电控发送时取反)
       pitch 电机配置 FEEDBACK_DIRECTION_REVERSE → PID内部自洽
```

## 五、控制架构 (STM32侧)

### 5.1 级联PID + 速度前馈

```
目标角度(deg) ─▶ [角度环PID] ─▶ 速度参考 + ff_vel(前馈) ─▶ [速度环PID] ─▶ 电压
                  ↑ IMU角度反馈            ↑ 陀螺仪角速度反馈
```

前馈数据来源:
1. NUC在协议帧中填了yaw_vel/pitch_vel (非零) → 直接使用
2. NUC没填 (全零) → STM32用连续两帧角度差分估算: `vel ≈ Δangle / Δt`, 经一阶低通滤波(alpha=0.4)

### 5.2 数据流

```
NUC帧 → serial_gimbal_cmd.c (解析+差分估算ff)
       → serial_cmd_data.yaw_vel / pitch_vel
       → robot_cmd.c SerialControlSet() 转发
       → gimbal_cmd_send.yaw_vel / pitch_vel (pubsub发布)
       → gimbal.c GimbalTask() 订阅
       → ff_yaw_vel / ff_pitch_vel (静态变量)
       → DJIMotorControl() 叠加到速度环ref
```

### 5.3 当前PID参数

**YAW GM6020:**

| 环 | Kp | Ki | Kd | MaxOut | 特殊 |
|----|----|----|----|---------|----|
| 角度环 | 8 | 0 | 0.3 | 350 deg/s | 微分先行+微分滤波(RC=0.005), 死区0.1° |
| 速度环 | 25 | 100 | 0 | 15000 | 积分限幅1500 |

**PITCH GM6020:**

| 环 | Kp | Ki | Kd | MaxOut | 特殊 |
|----|----|----|----|---------|----|
| 角度环 | 8 | 0 | 0.3 | 350 deg/s | 同YAW |
| 速度环 | 15 | 50 | 0 | 15000 | 增益比YAW低(之前大了会抖), 积分限幅800 |

### 5.4 前馈参数

| 参数 | 值 | 位置 | 说明 |
|------|----|------|------|
| 低通滤波alpha | 0.4 | serial_gimbal_cmd.c | 越大越快但越抖, 建议0.2~0.6 |
| 前馈限幅 | ±300 deg/s | serial_gimbal_cmd.c | 防止帧间角度跳变产生极大值 |
| dt有效范围 | 0.001s ~ 0.1s | serial_gimbal_cmd.c | 超出范围前馈置零 |

### 5.5 软件限位

| 轴 | 最大角度 | 最小角度 |
|----|---------|---------|
| YAW | +50° | -50° |
| PITCH | +35° | -35° |

## 八、联调流程

### Phase 1: 静态标定 (视觉单独)

1. 确认相机内参 (`camera.yaml`)
2. 确认激光瞄准点 (`control.yaml: u_L, v_L`)
3. 确认激光物理偏移 (`laser_offset_x, laser_offset_y`)

### Phase 2: 通信验证

1. NUC运行tracking程序, 终端输出 `DBG gimbal cmd: mode=1 yaw=xxx pitch=xxx`
2. 电控侧用J-Link Watch `serial_cmd_data.yaw` 和 `serial_cmd_data.pitch`, 应与NUC输出一致
3. 确认 `serial_cmd_data.mode != 0`

### Phase 3: 关前馈调基础PID

1. 注释 `gimbal.c` 中 ff 赋值行 (强制=0)
2. 遥控器控制, 观察是否平稳无震荡
3. NUC锁定静态目标, 确认稳态误差 < 0.5°
4. 调PID: 抖→降Kd; 响应慢→升Kp

### Phase 4: 开前馈调跟踪

1. 恢复ff赋值, NUC发追踪指令
2. 确认 `ff_yaw_vel` 方向正确、幅度合理 (慢速10~50, 快速100~300 deg/s)
3. 对比有/无前馈时 angle_PID.Err 峰值
4. 调alpha: 超调→降到0.2; 不够→升到0.6

### Phase 5: 远距离验证与延迟补偿

1. 20m/25m/30m距离逐步测试
2. 开启自适应增益 (kp_far=6.0)
3. 调 `system_delay_ms` 补偿延迟
4. 观察命中率, 启用自学习

### Phase 6: 安全验证

1. 拨轮急停 → 电机立即停止, 前馈清零
2. 拔NUC串口 → 电机保持当前位置, 不跳变
3. 重新插上 → 平滑恢复控制
4. NUC→遥控器切换 → 目标角度同步为当前IMU角度

## 九、比赛流程注意事项

### 赛前准备

- 三分钟准备阶段: 运算平台端放到指定台面, 传感器端固定到雷达基座
- 遥控器放到战场入口指定区域
- 电源管理模块必须外露, 主控接口外露不连接任何设备
- 航空线理顺收纳, 易于拉出连接

### 比赛中

- 云台仅在对方空中支援时上电
- 被判罚下: 激光+云台断电, 通信断开
- 激光检测模块离线: 视为被持续照射 (对方惩罚)

### 评分

| 评分项目 | 权重 |
|---------|------|
| 累计准确标注对方时间 | ×1 |
| **反制无人机次数** | **×50** |
| 成功解析信息波难度等级 | ×50 |

> 反制无人机权重极高(×50), 是系统的核心价值所在。

## 十、远距离精度提升优先级

| 优先级 | 措施 | 预期效果 |
|--------|------|----------|
| **1** | 换长焦镜头 (fx: 1000→2000) | 目标从1.7px→3.4px, 精度翻倍 |
| **2** | 提高检测分辨率 | 更多像素 → 更准定位 |
| **3** | 亚像素定位 (sub-pixel refinement) | 突破像素分辨率限制 |
| **4** | 延迟补偿 | 减少30ms跟踪滞后 |
| **5** | 速度前馈 (已实现) | 电机提前预判, 减少追踪延迟 |
| **6** | 自学习校准 | 越用越准 |

## 十一、常见问题速查

| 问题 | 排查方向 |
|------|----------|
| 云台上电不动 | 检查遥控器模式开关; 检查NUC mode是否=0 |
| 云台剧烈震荡 | 降低角度环Kp/Kd; 检查前馈是否过大 |
| 跟踪有明显滞后 | 开启前馈; 增大kp; 开启延迟补偿 |
| 远距离打不准 | 检查相机标定; 考虑换长焦镜头 |
| NUC断开后云台跳变 | 检查robot_cmd.c中NUC离线时的角度同步逻辑 |
| 前馈值始终为0 | Watch serial_cmd_data.yaw, 检查NUC是否每帧角度在变 |
| 前馈方向反 | 检查NUC角度正方向定义与IMU坐标系是否一致 |
| 编译后数据错乱 | robot_def.h改了结构体, 必须全部重编译(Clean+Build) |
